class pymerkle.validations.validation_receipt(*args, **kwargs)[source]¶

Bases: object

Encapsulates the result of proof validation

Parameters:
  • proof_uuid (str) – uuid of the validated proof (time-based)
  • proof_provider (str) – uuid of the Merkle-tree having provided the proof
  • result (bool) – Validation result (True iff the proof was found to be valid)

Instead of providing the above arguments corresponding to *args, a validation_receipt object may also be constructed in the following ways by employing **kwargs in order to load the JSON string of a given validation-receipt r:

>>> from pymerkle.valiation_receipts import validation_receipt
>>> s = validation_receipt(from_json=r.JsonString())
>>> t = validation_receipt(from_dict=json.loads(r.JsonString()))

Note

Constructing receipt in the above ways is a genuine replication, since the constructed receipts s and t have the same uuuid and timestamps as r

Variables:
  • header – (dict) Contains the keys uuid, timestamp, validation_moment
  • header.uuid – (str) uuid of the validation (time-based)
  • header.timestamp – (str) Validation moment (msecs) from the start of time
  • header.validation_moment – (str) Validation moment in human readable form
  • body – (dict) Contains the keys proof_uuid, proof_provider, result (see below)
  • body.proof_uuid – (str) See the homonymous argument of the constructor
  • body.proof_provider – (str) See the homonymous argument of the constructor
  • body.result – (bool) See the homonymous argument of the constructor
