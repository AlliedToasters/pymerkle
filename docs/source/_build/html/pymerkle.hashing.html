class pymerkle.hashing.hash_machine(hash_type='sha256', encoding='utf-8', security=True)[source]¶

Bases: object

Encapsulates the two basic hash utilities used accross this library.

Sole purpose of this class is to fix at construction the hash and encoding types used for encryption, so that these parameters need not be redefined every time a hash utility is invoked. Instances of this class are thus to be initialized with every new construction of a Merkle-tree or every time a proof validation is about to be performed

Parameters:
  • hash_type (str) – specifies the hash algorithm to be used by the machine; must be among the elements of the HASH_TYPES global variable (upper- or mixed-case with ‘-‘ instead of ‘_’ allowed). Defaults to sha256 if unspecified
  • encoding (str) – specifies the encoding algorithm to be used by machine before hashing; must be among the elements of the ENCODINGS global variable (upper- or mixed-case with - instead of _ allowed). Defaults to utf_8 if unspecified
  • security (bool) – defaults to True; plays role only if hash and encoding types are sha256 and utf-8 respectively. In this case, security standards are applied against second-preimage attack, i.e., single, resp. double arguments of the hash function will be prepended with 'x00', resp. 'x01' before hashing
Raises:

Exception – if hash_type, resp. encoding is not contained in HASH_TYPES, resp. ENCODINGS
Variables:
  • HASH_ALGORITHM – (builtin_function_or_method) Hash algorithm used by the machine. Defaults to SHA256.
  • ENCODING – (str) Encoding type used by the machine while hashing. Defaults to UTF-8.
  • SECURITY – (bool) Indicates that defense against second-preimage attack is activated (genuinely only for default values of hash and encoding types). Defaults to True.

multi_hash(signed_hashes, start)[source]¶

Hash utility used in proof validation

Repeatedly applies the core hash method over a tuple of signed hashes parenthesized in pairs as specified by accompanying signs

Parameters:
  • signed_hashes (tuple of (+1/-1, hash) pairs, where hash is of type str or bytes or bytearray) – a sequence of signed hashes
  • start (int) – position where the application of hash will start from
Returns:

a valid hexadecimal representing the computed hash
Return type:

str
Example: 
>>> from pymerkle.hashing import hash_machine
>>> machine = hash_machine()
>>> machine.multi_hash(((1, 'a'), (1, 'b'), (-1, 'c'), (-1, 'd')), 1)
'1453fd7ac377d0892731cfdbd60f656db6ed0d4f4475657335945634bb6d3243'

the result being equivalent to

hash(hash('a', hash('b', 'c')), 'd')

Note

Returns None if the inserted sequence of signed hashes was empty

Warning

When using this method, make sure that the combination of signs corresponds indeed to a valid parenthetization
